# Интеллектуальная система подбора персонала с семантическим анализом агрегированных данных

Система представляет собой многоагентную микросервисную платформу, предназначенную для автоматизированного подбора персонала. Основными функциями являются агрегация данных о кандидатах, парсинг резюме различных форматов, извлечение структурированной информации, генерация семантических представлений и выполнение гибридного поиска с учетом смысловой близости компетенций.


## Содержимое диска

В состав материалов дипломного проекта входят следующие файлы:

- hr-semantic-system.zip – архив, содержащий полный исходный код проекта (монорепозиторий)
- Презентация ДП.pptx – презентация дипломного проекта
- README.md – настоящий документ


## Руководство по сборке и запуску проекта

### 1. Требования

- Docker Desktop (версия 4.25 и выше) с включенным Docker Compose
- Минимум 8 ГБ оперативной памяти (рекомендуется 16 ГБ)
- Процессор с поддержкой AVX

### 2. Подготовка

1. Распакуйте архив `hr-semantic-system.zip`.
2. Перейдите в директорию проекта:
   ```bash
   cd hr-semantic-system
   ```
3. Перейдите в папку инфраструктуры:
   ```bash
   cd infra
   ```
4. Создайте файл переменных окружения:
   ```bash
   cp .env.example .env
   ```
   При необходимости отредактируйте `.env` (API-ключи, настройки баз данных и т.д.).

### 3. Запуск системы

```bash
# Из папки infra/
docker compose up --build -d
```

### 4. Проверка запуска

```bash
docker compose ps
```

После успешного запуска основные сервисы доступны по адресам:

- Пользовательский интерфейс: http://localhost:3000
- Orchestrator Service: http://localhost:8004/docs
- Extraction Service: http://localhost:8001/docs
- Storage Service: http://localhost:8002/docs
- Search Service: http://localhost:8003/docs

### 5. Остановка системы

```bash
# Остановка без удаления данных
docker compose down

# Полная очистка (включая volumes)
docker compose down -v
```



## Руководство пользователя

### Основные сценарии работы с системой

После запуска системы пользователь (рекрутер) может выполнять следующие действия через веб-интерфейс:

1. **Авторизация**
   - Перейти по адресу `http://localhost:3000`
   - Войти под учетными данными (стартовый аккаунт администратора: `admin` + `admin123`)

2. **Загрузка резюме**
   - Перейти в раздел «Загрузка резюме»
   - Выбрать один или несколько файлов (PDF, DOCX, PNG, JPG)
   - Нажать кнопку загрузки
   - Система автоматически выполнит парсинг, извлечение сущностей и сохранение в векторное хранилище

3. **Поиск кандидатов**
   - Перейти в раздел «Поиск»
   - Ввести описание вакансии на естественном языке (например: «Senior Python разработчик с опытом FastAPI и PostgreSQL»)
   - При необходимости задать дополнительные фильтры (опыт, зарплата, локация, навыки)
   - Нажать «Найти»
   - Получить ранжированный список кандидатов с объяснением релевантности

4. **Просмотр профиля кандидата**
   - Кликнуть на карточку кандидата в результатах поиска
   - Откроется детальная информация: структурированный профиль, опыт работы, навыки, семантические совпадения и исходный документ

### Работа через API (для разработчиков)

**Пример загрузки резюме:**
```bash
curl -X POST http://localhost:8004/orchestrate/upload \
  -H "Authorization: Bearer <JWT_TOKEN>" \
  -H "X-Correlation-ID: upload-001" \
  -F "file=@candidate_resume.pdf"
```

**Пример поиска кандидатов:**
```bash
curl -X POST http://localhost:8004/orchestrate/search \
  -H "Authorization: Bearer <JWT_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "Python backend разработчик с опытом машинного обучения",
    "top_k": 10,
    "use_rerank": true
  }'
```

### Возможные проблемы и решения

- **Сервис долго запускается** — особенно `search-service` и `keycloak` (до 3 минут при первом запуске).
- **Ошибка авторизации** — убедитесь, что Keycloak запущен.
- **Не удаётся загрузить файл** — максимальный размер 20 МБ. Поддерживаемые форматы: PDF, DOCX, PNG, JPG.
- **Поиск возвращает мало результатов** — попробуйте более общий запрос или проверьте наличие кандидатов в базе.
